# Code of Conduct

## Our Pledge

This project is a local-only security learning lab. Contributors and
maintainers are expected to keep the project respectful, constructive, and safe
for learners with different experience levels.

We pledge to:

- be welcoming and patient with good-faith questions
- give feedback that is specific, kind, and actionable
- respect different backgrounds, skill levels, and viewpoints
- keep security discussion scoped to defensive, authorized, and local lab use
- avoid harassment, personal attacks, or exclusionary behavior

## Expected Behavior

Examples of behavior that helps this project:

- explain security concepts without shaming beginners
- separate intentional lab vulnerabilities from accidental project issues
- document assumptions, safety boundaries, and reproduction steps clearly
- report accidental security issues privately when appropriate
- keep pull requests focused and reviewable

## Unacceptable Behavior

Examples of behavior that is not acceptable:

- harassment, insults, threats, or discriminatory language
- encouraging unauthorized testing of third-party systems
- adding real credentials, real targets, or public exploit instructions
- repeatedly derailing issues or pull requests
- publishing accidental security issues before maintainers can review them

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other
contributions that violate this code of conduct. Maintainers may also block or
ban contributors who repeatedly or seriously violate these expectations.

If you need to report a conduct concern, contact the repository maintainer
privately and include relevant context, links, and screenshots where helpful.

## Scope

This code of conduct applies to project spaces such as issues, pull requests,
discussions, documentation, and other collaboration connected to this
repository.
